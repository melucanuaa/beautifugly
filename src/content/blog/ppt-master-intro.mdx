## 作者是谁

何雨果（Hugo He），投融资从业者，不是程序员。日常就是做 PPT、审 PPT、改 PPT。他自己的说法：**"我是最挑剔的用户。"**

这个项目不是程序员觉得"这个需求有意思"然后做的，是一个每天被 PPT 折磨的打工人，花了大量时间打磨出来的工具。这种出身决定了它在"到底什么样的 PPT 才算能用"这个问题上，判断比大多数 AI 工具都准。

## PPT Master 的第四条路

PPT Master 走了一条完全不同的技术路线：**AI 生成 SVG，脚本将 SVG 转换为 DrawingML。**

<div style="background:var(--color-bg-card);border-radius:10px;padding:14px;margin:1rem 0;font-size:0.82rem;border-left:3px solid var(--color-accent)">
  <strong>为什么这条路走得通？</strong> SVG 和 DrawingML 本质上是同一类东西——都是基于绝对坐标的 2D 矢量格式。矩形、路径、渐变、阴影的概念一一对应。转换是"方言翻译"，不是格式代沟。
</div>

这意味着 AI 负责它擅长的事（排版、配色、信息架构），脚本负责精确的格式转换。最终输出的 PPTX 里，**每个形状、文本框、渐变、阴影都是原生 PowerPoint 对象**——点击就能编辑，不是截图，不是图片化文本。

## 跟竞品到底差多少

| 对比项 | PPT Master | Gamma | WPS AI | Copilot PPT |

## 不适合的场景

  <strong>注意：</strong> PPT Master 的定位是"AI 是你的设计师，不是完工师"。它帮你消除 90% 从零开始的工作量，但最后 10% 的精细化调整——排版微调、图片替换、内容校对——还是需要你在 PowerPoint 里手动完成。如果你期望的是"一键出成品，完全不用改"，那这不是你要的工具。

## 快速上手

只需要三步：

<div style="background:var(--color-bg-card);border-radius:10px;padding:14px;margin:1rem 0;font-size:0.82rem;border-left:3px solid var(--color-accent)">
  <strong>1.</strong> 安装 Python 3.10+ 环境<br/>
  <strong>2.</strong> 克隆仓库并安装依赖：<code style="background:var(--color-bg);padding:2px 6px;border-radius:4px;font-size:0.78rem">git clone https://github.com/hugohe3/ppt-master.git</code><br/>
  <strong>3.</strong> 在 Claude Code / Cursor / VS Code Copilot 中打开项目，对着你的文档说"帮我生成一份 PPT"

支持批量生成——通过 Shell 或 Python 脚本调用核心脚本，可以实现企业级批量文档转 PPT 的自动化。

## 最后

PPT Master 最打动我的不是技术多牛，而是它的出身。一个每天做 PPT 的打工人，被折磨到自己动手，做出来的东西自然比"程序员觉得用户需要"的工具更靠谱。

技术上，SVG 转 DrawingML 这条路确实是目前 AI PPT 领域最务实的解法——既保留了 AI 的排版能力，又保证了输出的可编辑性。MIT 协议、全程本地化、不绑定平台，对打工人来说基本没有使用门槛。

如果你也被 PPT 折磨过，值得试试。
