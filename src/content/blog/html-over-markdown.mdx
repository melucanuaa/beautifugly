最近看到一条推文（750 万浏览），来自 Anthropic 的 Thariq——他目前在做 Claude Code。核心观点：**用 AI 工作时，让输出 HTML 而不是 Markdown，效果好得离谱。**

## 先说结论

<table>
  <thead>
    <tr>
      <th>对比项</th>
      <th>Markdown</th>
      <th>HTML</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>信息密度</strong></td>
      <td>纯文本，最多加个表格</td>
      <td>表格、图表、SVG、交互控件，什么都能放</td>
    </tr>
    <tr>
      <td><strong>可读性</strong></td>
      <td>超过 100 行就没人想看</td>
      <td>标签页、折叠块、高亮，扫一眼就懂</td>
    </tr>
    <tr>
      <td><strong>可分享性</strong></td>
      <td>得用渲染器才能看</td>
      <td>浏览器直接打开，发个链接就行</td>
    </tr>
    <tr>
      <td><strong>交互性</strong></td>
      <td>没有</td>
      <td>滑块、按钮、实时预览，调完参数复制回 AI</td>
    </tr>
    <tr>
      <td><strong>生成速度</strong></td>
      <td>快</td>
      <td>慢 2-4x，但结果值得</td>
    </tr>
  </tbody>
</table>

## 为什么 HTML 更好

<div style="display:grid;grid-template-columns:repeat(2,1fr);gap:12px;margin:1rem 0">
  <div style="background:var(--color-bg-card);border-radius:10px;padding:14px;border-left:3px solid var(--color-accent)">
    <div style="font-weight:600;font-size:0.85rem;margin-bottom:6px">信息密度</div>
    <div style="font-size:0.8rem;color:var(--color-text-light)">Markdown 只有标题、列表、表格、代码块。HTML 能做对比表格、SVG 图表、代码高亮、交互控件。</div>
  </div>
  <div style="background:var(--color-bg-card);border-radius:10px;padding:14px;border-left:3px solid var(--color-accent)">
    <div style="font-weight:600;font-size:0.85rem;margin-bottom:6px">你会读 HTML，但不会读 Markdown</div>
    <div style="font-size:0.8rem;color:var(--color-text-light)">超过 100 行的 Markdown 没人读。HTML 有标签页、折叠块、视觉层级——500 行文档两三屏看完。</div>
  </div>
  <div style="background:var(--color-bg-card);border-radius:10px;padding:14px;border-left:3px solid var(--color-accent)">
    <div style="font-weight:600;font-size:0.85rem;margin-bottom:6px">分享方便</div>
    <div style="font-size:0.8rem;color:var(--color-text-light)">Markdown 需要渲染器。HTML 丢到任何地方，浏览器打开就能看。上传 S3 发链接即可。</div>
  </div>
  <div style="background:var(--color-bg-card);border-radius:10px;padding:14px;border-left:3px solid var(--color-accent)">
    <div style="font-weight:600;font-size:0.85rem;margin-bottom:6px">交互性</div>
    <div style="font-size:0.8rem;color:var(--color-text-light)">滑块调参数、下拉切换视图、点按钮复制结果。AI → 人调整 → 喂回 AI，人始终在回路里。</div>
  </div>
</div>

Thariq 原话："没有 HTML 的时候，Claude 会用 Unicode 字符'估算颜色'来画图。"

## 唯一的缺点

<div style="background:var(--color-bg-card);border-radius:10px;padding:14px;margin:1rem 0;font-size:0.82rem;border-left:3px solid var(--color-accent-pink)">
  <strong>版本控制</strong>：HTML 的 Git diff 很难读，比 Markdown 差很多。如果你需要对生成内容做版本控制，Markdown 还是有优势。
</div>

## 我的看法

Thariq 最后说了一句很打动我的话：

> "用 HTML 之后，我比以前任何时候都更'在回路里'——与其让 Claude 自己做决定，HTML 让人始终保持参与。"

这和 Vibe Coding 的理念很像。AI 是工具，不是替代品。你需要能看懂它在做什么、能调它做出来的东西、能把它输出的结果继续喂回去。HTML 恰好提供了这种能力。

**试试看吧，下次让 AI 输出 HTML。**
