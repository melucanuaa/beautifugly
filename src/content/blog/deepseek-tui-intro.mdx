<p>最近 GitHub 上冲出一个狠角色——<strong>DeepSeek-TUI</strong>，两天涨了 3500 stars，目前总星标已突破 23K，一度登上热榜第一。</p>

<p>它是什么？一句话：<strong>把 DeepSeek V4 模型塞进终端，做成一个能读写文件、执行命令、管理 Git 的编程 Agent。</strong></p>

<p>听起来像 Claude Code？没错，但它是开源的，跑在 DeepSeek 的模型上，而且便宜得多。</p>

<h2>为什么值得关注</h2>

<p>先看几个关键数据：</p>

<ul>
  <li><strong>23,363 stars</strong>，1,884 forks，30 位贡献者</li>
  <li>基于 <strong>DeepSeek V4</strong>（deepseek-v4-pro / deepseek-v4-flash）</li>
  <li>支持 <strong>100 万 token</strong> 上下文窗口</li>
  <li>MIT 协议，完全开源</li>
  <li>用 <strong>Rust</strong> 写的，跨平台（Linux / macOS / Windows）</li>
  <li>最新版本 v0.8.25，更新非常频繁</li>
</ul>

<p>开发者 Hunter Bown 是个有意思的人——本硕读的是音乐教育，目前在法学院读书，不是科班程序员。这个项目 2026 年 1 月发布，4 月底 DeepSeek V4 升级后彻底爆了。</p>

<h2>核心功能一览</h2>

<p>初学者建议从 Agent 模式开始，熟悉了再切 YOLO。</p>

<h3>自动模式（Auto Mode）</h3>

<p>这是 DeepSeek-TUI 的杀手锏。输入 <code>/model auto</code>，它会自动决定每一轮对话用哪个模型、开多大推理强度：</p>

<ul>
  <li>简单问题 → <code>deepseek-v4-flash</code>，不开推理，便宜快速</li>
  <li>复杂编码/调试 → <code>deepseek-v4-pro</code>，开高推理，质量优先</li>
</ul>

<p>背后是一个轻量路由调用，先用 flash 模型判断任务难度，再分配资源。你不用操心选模型，专注干活就行。</p>

<h3>推理强度切换</h3>

<p>按 <code>Shift+Tab</code> 可以循环切换推理等级：<code>off → high → max</code>。</p>

<ul>
  <li><strong>off</strong>：不推理，直接回答，适合简单问答</li>
  <li><strong>high</strong>：中等推理，适合一般编码任务</li>
  <li><strong>max</strong>：最强推理，适合架构设计、复杂调试</li>
</ul>

<h3>完整工具集</h3>

<p>DeepSeek-TUI 不只是聊天，它有一套完整的工具：</p>

<ul>
  <li><strong>文件操作</strong>：读写本地文件</li>
  <li><strong>Shell 执行</strong>：运行终端命令</li>
  <li><strong>Git 管理</strong>：提交、分支、diff</li>
  <li><strong>网页搜索</strong>：实时获取信息</li>
  <li><strong>子 Agent</strong>：同时启动 1-16 个 flash 子 Agent 并行工作</li>
  <li><strong>MCP 协议</strong>：连接外部工具扩展</li>
  <li><strong>LSP 诊断</strong>：编辑代码后实时显示错误和警告</li>
</ul>

<h3>工作区回滚</h3>

<p>这个设计很贴心——每轮操作前后自动生成快照，用 <code>/restore</code> 或 <code>revert_turn</code> 就能回滚，不会动你项目的 <code>.git</code>。</p>

<p>相当于给你的代码加了一个"后悔药"。</p>

<h2>安装体验</h2>

<p>安装非常简单，一行命令：</p>

```
npm install -g deepseek-tui
```

<p>首次运行会提示输入 DeepSeek API Key，保存到 <code>~/.deepseek/config.toml</code>。</p>

```
deepseek --version
deepseek --model auto
```

<p>如果你在大陆，npm 下载慢，可以用 Cargo 镜像：</p>

```
# ~/.cargo/config.toml
# [source.crates-io]
# replace-with = "tuna"
# [source.tuna]
# registry = "sparse+https://mirrors.tuna.tsinghua.edu.cn/crates.io-index/"

cargo install deepseek-tui-cli --locked
cargo install deepseek-tui --locked
```

<p>也可以直接从 GitHub Releases 下载预编译二进制。</p>

<h2>实际使用感受</h2>

<h3>速度</h3>

<p>DeepSeek V4 的响应速度很快，尤其是 <code>deepseek-v4-flash</code>，几乎感觉不到延迟。Pro 模型稍慢一些，但推理质量明显更高。</p>

<h3>上下文理解</h3>

<p>支持 100 万 token 上下文，实际使用中在大型项目里也不会"失忆"。上下文快满时会自动压缩，还能看到前缀缓存的命中率。</p>

<h3>代码质量</h3>

<p>生成的代码质量不错，尤其是配合 LSP 诊断——每次编辑后会自动检查语法错误，把诊断信息送入模型上下文，下一步推理就更准。</p>

<h3>中文支持</h3>

<p>UI 支持简体中文，自动检测系统语言。文档也有完整的中文 README。</p>

<h2>一些不足</h2>

<ul>
  <li>项目还在快速迭代，版本号还是 0.8.x，API 和行为可能变化</li>
  <li>依赖 DeepSeek 官方 API，国内网络环境可能需要代理</li>
  <li>社区生态不如 Claude Code 成熟（没有 everything-claude-code 那样的 100K stars 配套项目）</li>
  <li>长时间任务的缓存命中率会下降，成本优势会减弱</li>
</ul>

<h2>总结</h2>

<p>DeepSeek-TUI 代表了一个趋势：<strong>AI 编程工具正在从"昂贵的闭源服务"走向"便宜的开源基础设施"</strong>。</p>

<p>它不是 Claude Code 的简单替代品，而是一个有自己特色的工具——超长上下文、子 Agent 并行、自动模式、工作区回滚，这些功能在某些场景下比 Claude Code 更好用。</p>

<p>如果你是终端党，或者对 AI 编程工具的成本敏感，值得试试。</p>
